Implement src/wssec/username_token.rs:

PasswordDigest formula — implement EXACTLY from OASIS spec (rpos reference):
```rust
use sha1::{Sha1, Digest};
use base64::engine::general_purpose::STANDARD as BASE64;
use base64::Engine;

pub fn compute_digest(nonce_b64: &str, created: &str, password: &str) -> Result<String, SoapFault> {
    let nonce_bytes = BASE64.decode(nonce_b64)
        .map_err(|_| SoapFault::sender("Invalid nonce encoding"))?;
    let mut hasher = Sha1::new();
    hasher.update(&nonce_bytes);
    hasher.update(created.as_bytes());
    hasher.update(password.as_bytes());
    Ok(BASE64.encode(hasher.finalize()))
}
```

Key struct for parsed UsernameToken:
```rust
pub struct UsernameToken {
    pub username: String,
    pub password: String,
    pub password_type: PasswordType,
    pub nonce: Option<String>,
    pub created: Option<String>,
}

pub enum PasswordType {
    Digest,  // PasswordDigest
    Text,    // PasswordText
}
```

parse_username_token(xml_bytes: &[u8]) → Result&lt;UsernameToken, SoapFault&gt;:
- Use quick-xml to parse the wsse:Security header bytes
- Extract wsse:UsernameToken element and its children
- wsse namespace: `http://docs.oasis-open.org/wss/2004/01/oasis-200401-wss-wssecurity-secext-1.0.xsd`
- wsu namespace: `http://docs.oasis-open.org/wss/2004/01/oasis-200401-wss-wssecurity-utility-1.0.xsd`
- Password Type attribute for PasswordDigest: `http://docs.oasis-open.org/wss/2004/01/oasis-200401-wss-username-token-profile-1.0#PasswordDigest`
- Password Type attribute for PasswordText: `http://docs.oasis-open.org/wss/2004/01/oasis-200401-wss-username-token-profile-1.0#PasswordText`

validate_username_token(
    security_bytes: &[u8],
    get_password: &dyn Fn(&str) -> Option&lt;String&gt;,
    nonce_cache: &mut RotatingNonceCache,
    tolerance_secs: i64,
    now: chrono::DateTime&lt;chrono::Utc&gt;,

Public interface:
```rust
/// Pass 1: parse WSDL bytes into WsdlDefinition with unresolved QName strings.
/// Called with raw WSDL bytes (file contents or embedded bytes).
pub fn parse_wsdl(bytes: &[u8]) -> Result<WsdlDefinition, WsdlError>
```

Internal functions (one per WSDL element type):
- parse_service(node) → Service
- parse_port(node) → Port (binding is a QName string resolved in pass 2)
- parse_port_type(node) → PortType
- parse_operation(node) → Operation
- parse_binding(node) → Binding
- parse_binding_operation(node) → BindingOperation
- parse_message(node) → Message
- parse_message_part(node) → MessagePart

WSDL 1.1 namespace: `http://schemas.xmlsoap.org/wsdl/`
SOAP 1.1 binding namespace: `http://schemas.xmlsoap.org/wsdl/soap/`
SOAP 1.2 binding namespace: `http://schemas.xmlsoap.org/wsdl/soap12/`

Detecting SOAP version in binding: check the namespace of the soap:binding element. If `http://schemas.xmlsoap.org/wsdl/soap12/` → SoapVersion::Soap12. If `http://schemas.xmlsoap.org/wsdl/soap/` → SoapVersion::Soap11.

For inline xs:schema inside wsdl:types: record the schema node's XML bytes using roxmltree. The resolver will pass these bytes to xsd::parse_schema.

Store raw QName strings with their prefix context so the resolver can expand them to full namespace URIs. Use a helper function `resolve_qname(prefix: Option<&str>, local: &str, node: &Node) -> QName` that calls `node.lookup_namespace_uri(prefix)`.

Define WsdlError with thiserror variants: MalformedXml(String), MissingAttribute(String), UnknownNamespace(String).
