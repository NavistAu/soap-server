Implement src/xsd/parser.rs following the python-zeep xsd/visitor.py pattern. The public interface is:

```rust
use roxmltree::Node;
use crate::qname::QName;
use crate::xsd::types::*;
use crate::xsd::elements::*;

/// Pass 1 result: all types and elements collected from one XSD schema node.
/// QName references are stored as strings — resolution happens in pass 2.
pub struct RawSchema {
    pub target_namespace: Option<String>,
    pub types: HashMap<QName, XsdType>,
    pub elements: HashMap<QName, XsdElement>,
    pub attribute_groups: HashMap<QName, AttributeGroup>,
    pub groups: HashMap<QName, Group>,
    pub imports: Vec<SchemaImport>,
    pub includes: Vec<SchemaInclude>,
}

pub struct SchemaImport {
    pub namespace: Option<String>,
    pub schema_location: Option<String>,
}

pub struct SchemaInclude {
    pub schema_location: String,
}

/// Entry point: parse a <xs:schema> node (from roxmltree).
pub fn parse_schema(node: Node<'_, '_>) -> Result<RawSchema, SchemaError> { ... }
```

Implement visit functions for each XSD construct. Key namespace constants:
- XS namespace: `http://www.w3.org/2001/XMLSchema`
- Check element tag by: `node.tag_name().namespace() == Some(XS_NS) && node.tag_name().name() == "complexType"`

CRITICAL: QName resolution during pass 1 — when you encounter `type="tns:Foo"`, split on ":" to get prefix and local name, then look up the prefix in the current node's namespace mappings: `node.lookup_namespace_uri("tns")`. Store the resolved namespace URI + local name as a QName. If no prefix (e.g., `type="string"` without a prefix but in the XS namespace), resolve using the default namespace.

Port these functions from python-zeep xsd/visitor.py:
- visit_complex_type(node) → XsdType::Complex(ComplexType)
- visit_simple_type(node) → XsdType::Simple(SimpleType)
- visit_sequence(node) → Vec&lt;XsdElement&gt;
- visit_all(node) → Vec&lt;XsdElement&gt;
- visit_choice(node) → Vec&lt;XsdElement&gt;
- visit_element(node) → XsdElement
- visit_attribute(node) → XsdAttribute
- visit_attribute_group_def(node) → AttributeGroup
- visit_group_def(node) → Group
- visit_any(node) → AnyElement
- visit_restriction(node) → Restriction
- visit_extension(node) → (QName, ComplexContent) [base + extended content]
- visit_list(node) → ListDef
- visit_union(node) → UnionDef

Parse all facets in restriction: xs:enumeration, xs:minInclusive, xs:maxInclusive, xs:minExclusive, xs:maxExclusive, xs:length, xs:minLength, xs:maxLength, xs:pattern, xs:whiteSpace, xs:totalDigits, xs:fractionDigits.

For xs:import and xs:include: collect into RawSchema.imports / RawSchema.includes for the resolver to load in pass 2.

Define a SchemaError type using thiserror. Key variants: MalformedXml(String), UnknownRef(String), CycleDetected(String).

Update src/xsd/mod.rs to pub use parser::parse_schema.

Write inline tests that parse minimal XSD snippets (not the full fixture — fixture tests go in integration tests). Each test constructs a valid XSD string, calls parse_schema, and asserts the output structure.

IMPORTANT — state machine awareness: roxmltree parse requires the full document bytes. For inline schema nodes within a WSDL, the caller (wsdl::resolver) will extract the schema node and pass it to parse_schema. The parser does not load files — that is the resolver's job.
