```rust
/// SchemaLoader trait — abstracts file I/O for testability.
/// The wsdl::resolver provides a FileSchemaLoader impl; tests use MockSchemaLoader.
pub trait SchemaLoader: Send + Sync {
    fn load(&self, namespace: Option<&str>, location: &str) -> Result<String, SchemaError>;
}

/// Resolved schema: all types and elements with references fully resolved.
pub struct ResolvedSchema {
    pub target_namespace: Option<String>,
    pub types: HashMap<QName, ComplexType or SimpleType>,  // resolved, no ComplexExtension variants
    pub elements: HashMap<QName, ResolvedElement>,
}

/// Entry point: resolve a RawSchema (and its imports) into a TypeRegistry.
pub fn resolve_schema(
    raw: RawSchema,
    loader: &dyn SchemaLoader,
    already_loaded: &mut HashMap<String, RawSchema>,  // cycle guard keyed by location
) -> Result<TypeRegistry, SchemaError> { ... }
```

Resolution algorithm (ported from python-zeep xsd/types/complex.py):
1. Collect all RawSchemas (root + imported/included, recursively loaded via SchemaLoader, using already_loaded as visited set)
2. Merge all into one flat type map keyed by QName
3. For each type in dependency order (resolve leaves first):
   - If ComplexExtension: resolve base type first (recursive), then prepend base.all_elements to child elements
   - If ComplexRestriction: resolve base type, then apply restriction content (replace content model)
   - Track in-progress types in a `resolving: HashSet&lt;QName&gt;` for cycle detection
   - Mark completed types (memoization) to avoid re-resolving

Group and AttributeGroup resolution:
- When xs:group ref= appears in content: look up Group by QName, inline its content
- When xs:attributeGroup ref= appears: look up AttributeGroup, merge its attributes
