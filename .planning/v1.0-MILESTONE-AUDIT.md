### New Finding: Address Rewrite Semantic Defect

| Connection | Status | Issue |
|-----------|--------|-------|
| `wsdl_get_handler → rewrite_wsdl_address` in multi-service mode | **PARTIAL** | `src/server.rs:744` uses `svc.mount_path` ("/soap") instead of per-service path ("/soap/a") for address rewrite |

**Impact:** WSDL returned by `GET /soap/a?wsdl` contains `soap:address location="http://host/soap"` instead of `http://host/soap/a`. A SOAP client reading this WSDL would POST to the wrong path.

**Severity:** Non-blocking semantic defect. Single-service mode (primary use case) works correctly. No automated test exercises this path.

**Fix path:** `wsdl_get_handler` needs the per-service path — either via axum's `MatchedPath` extractor or by storing per-service paths in the handler state.
