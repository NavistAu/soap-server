### Integration Gap: WSDL GET in Multi-Service Mode

- **Location:** `src/server.rs` lines 399-412
- **Issue:** `into_router()` multi-service branch registers only POST routes per service path. No GET handler is added for WSDL serving.
- **Impact:** `GET /soap/a?wsdl` returns 405 in multi-service mode
- **Affected requirements:** WSDL-04 (partially — single-service mode works correctly)
- **Severity:** Non-blocking (WSDL-04 requirement does not specify multi-service; single-service GET ?wsdl works)
